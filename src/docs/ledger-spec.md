# The Intent-to-Asset Translation Ledger
> Business rationale and observability strategy — stakeholder reference.

---

## The Problem: The Rationale Void

Traditional developer logs record the mutation of code state (what changed) but drop the non-deterministic contextual rationale (why it changed). This creates an acute organizational vulnerability when multi-agent systems generate and execute code changes on the fly.

## The Operational Translation Rule

To bridge the deep cognitive translation gap between the rushing Tech Lead, the non-technical operator, and the corporate auditor, the interface completely discards transient raw terminal streams. It establishes a permanent, traceable chain of custody by binding human intent directly to variable infrastructure costs and machine hypotheses.

---

## The Six System Vectors (What Every Ledger Row Must Display)

| Vector | Description |
|---|---|
| **Human Intent Anchor** | The original natural language prompt string typed by the human operator that initiated the downstream workflow execution graph. |
| **Machine Assumption Log** | A clean, plain-English serialized logic statement detailing the exact hypothesis the agent formulated to justify its chosen execution path. |
| **Bracketed Technical Trace** | High-precision telemetry data in fixed-width monospace: model tier, token volume, token velocity. |
| **Real-Time Financial Attribution** | Variable COGS + AER: `AER = Attributed Revenue / Compute Cost` |
| **Intent Drift Variance (%)** | Structural alert metric tracking how far the agent's sub-task decisions have deviated from the original master intent string over long execution loops. |
| **Context Window Gas Gauge** | Running visual capacity metric showing how close the agent's runtime thread memory is to breaching its model's context threshold limit. |

---

## The Three Core Intercept Controls

| Control | Behavior |
|---|---|
| **Global Kill-Switch** | Prominent emergency stop widget. Flushes React state engine, forces hard shutdown across all executing agent threads. |
| **Circuit-Breaking Human Gate** | Automatically traps high-variance code modifications or high-cost loops, forcing affected agent into flashing Amber-coded [PAUSED] state. |
| **State Expiry Review Timer** | Visual countdown embedded in trapped anomaly cards (e.g., "Auto-Aborting in 02:45"). Automatically kills thread if operator does not respond — prevents open loop connection timeouts. |
